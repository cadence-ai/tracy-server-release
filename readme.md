# Tracy Server

Tracy is a self-hosted AI backend server built on [Ollama](https://ollama.com) and [Qdrant](https://qdrant.tech). It exposes a REST API to manage language models, vector collections, agents, conversations, and document extraction — all running locally via Docker.

---

## Requirements

- [Docker](https://docs.docker.com/get-docker/) with Compose plugin
- NVIDIA GPU + [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html) *(required for GPU acceleration)*
- PowerShell 7+ *(for the deploy script on Windows)*

---

## Installation

### 1. Configure

Before the first launch, fill in `config/config.json`:

```json
{
  "token": "",
  "admin_token": ""
}
```

| Field | Description |
|---|---|
| `token` | Access token — required in the `Authorization: Bearer <token>` header for every API call |
| `admin_token` | License token obtained at account creation on the Tracy platform |

> **Note:** The `token` field is auto-generated by the deploy script (see below). You only need to fill it manually if you deploy without the script.

---

### 2. Deploy

#### Windows (recommended)

Run the PowerShell deploy script from the project root:

```powershell
.\deploy.ps1
```

The script will:
1. Generate a cryptographically secure random token and write it to `config/config.json`
2. Stop any running containers (`docker compose down`)
3. Rebuild the Tracy image from scratch (`docker compose build --no-cache tracy`)
4. Start all services in detached mode (`docker compose up -d`)
5. Stream Tracy's logs (`docker logs tracy --follow`)

The generated token is printed in the terminal — save it, you will need it to authenticate.

#### Linux / macOS

Generate a token manually, then start the stack:

```bash
# Generate a token
TOKEN=$(openssl rand -base64 32)
echo "Token: $TOKEN"

# Write it to the config file
jq --arg t "$TOKEN" '.token = $t' config/config.json > tmp.json && mv tmp.json config/config.json

# Start
docker compose down
docker compose build --no-cache tracy
docker compose up -d
docker logs tracy --follow
```

## Services

The stack starts three containers on a shared `tracy_net` bridge network:

| Service | Image | Port | Description |
|---|---|---|---|
| `ollama` | `ollama/ollama:latest` | `11434` | Local LLM runtime (GPU-accelerated) |
| `qdrant` | `qdrant/qdrant:latest` | `6333` / `6334` | Vector database for embeddings |
| `tracy` | `tracy/tracy` (local build) | `9090` *(default)* | Tracy REST API server |

The `PORT` and `BASE_URL` values can be overridden via environment variables:

```bash
PORT=8080 BASE_URL=http://my-server:8080 docker compose up -d
```

Data is persisted in three named Docker volumes: `ollama_data`, `qdrant_data`, and `tracy_database`.

## Authentication

All endpoints except `/` and `/docs/*` require a Bearer token:

```
Authorization: Bearer <your_token>
```

Requests without a valid token return `401 Unauthorized` or `403 Forbidden`.

## API Reference

Base URL: `http://localhost:9090` *(or your configured `BASE_URL`)*

Interactive documentation is available at [`/docs/`](http://localhost:9090/docs/).

---

### Status

| Method | Path | Description |
|---|---|---|
| `GET` | `/` | Health check — returns `Tracy is running` |

---

### Models

Manage Ollama models available to Tracy.

| Method | Path | Description |
|---|---|---|
| `POST` | `/model/download` | Download a model from Ollama |
| `DELETE` | `/model` | Delete a model |
| `GET` | `/model` | List all available models |

---

### Collections

Collections are vector namespaces used to store and search document embeddings.

| Method | Path | Description |
|---|---|---|
| `POST` | `/collection` | Create a new collection |
| `DELETE` | `/collection` | Delete a collection |
| `GET` | `/collection` | List all collections |

---

### Files

Upload and manage files within a collection. Files are parsed, chunked, and embedded automatically.

| Method | Path | Description |
|---|---|---|
| `POST` | `/collection/file` | Upload a file into a collection |
| `DELETE` | `/collection/file` | Delete a file from a collection |
| `GET` | `/collection/file` | List files in a collection |

---

### Agents

Agents are configurable AI personas bound to a model and optionally a collection.

| Method | Path | Description |
|---|---|---|
| `GET` | `/agent` | List all agents |
| `GET` | `/agent/name` | Get an agent by name |
| `POST` | `/agent/agent` | Create a standard agent |
| `POST` | `/agent/extractor` | Create an extractor agent |
| `DELETE` | `/agent` | Delete an agent |

---

### Conversations

Conversations maintain message history for a given agent session.

| Method | Path | Description |
|---|---|---|
| `GET` | `/conversation` | List all conversations |
| `POST` | `/conversation` | Create a new conversation |
| `DELETE` | `/conversation` | Delete a conversation |

---

### Messages

| Method | Path | Description |
|---|---|---|
| `GET` | `/message` | List messages in a conversation |

---

### Chat

Send a message and receive a response as a **Server-Sent Events (SSE)** stream.

| Method | Path | Description |
|---|---|---|
| `POST` | `/chat` | Send a chat message (streaming SSE response) |

---

### Extraction

Run a structured extraction using an extractor agent.

| Method | Path | Description |
|---|---|---|
| `POST` | `/extract` | Extract structured data from input |

## Project Structure

```
.
├── config/
│   └── config.json       # Authentication tokens
├── ignore/               # Ignore rules for file ingestion
├── static/               # API documentation (served at /docs/)
├── exe/                  # Bundled executables (antiword, etc.)
├── TracyServer           # Compiled binary
├── TracyServer.sha256    # Binary checksum
├── .sql                  # Database schema
├── Dockerfile
├── docker-compose.yml
└── deploy.ps1
```

## Volumes & Data

| Volume | Mount path | Contents |
|---|---|---|
| `ollama_data` | `/root/.ollama` | Downloaded Ollama models |
| `qdrant_data` | `/qdrant/storage` | Vector index data |
